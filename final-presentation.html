Background

배경과 동기

  • 기존 뽀모도로는 보통 25분 집중, 5분 휴식이라는 고정 루틴을 사용한다.
  • 하지만 코딩, 암기, 글쓰기, 시험공부처럼 작업마다 필요한 집중 방식이 다르다.
  • 사용자마다 집중 성공 패턴도 다르기 때문에 같은 시간표를 모두에게 적용하기 어렵다.
  • Focus Orbit은 작업과 기록에 맞춰 변화하는 타이머를 목표로 한다.

Problem

기존 뽀모도로의 한계

고정 시간

모든 작업에 동일한 집중/휴식 시간이 적용된다.

단발성 흐름

반복 세션 관리가 약하면 실제 뽀모도로 흐름이 흐려진다.

상태 전달 부족

사용자가 다른 화면을 보는 동안 타이머 상태를 놓치기 쉽다.

기록 활용 부족

완료와 중단 기록이 다음 루틴에 충분히 반영되지 않는다.

Reference

참고자료 기반 적응성 설명

  • 참고자료는 특정 시간이 절대적으로 최적이라는 근거가 아니라, 고정 루틴의 한계를 설명하는 근거로 활용했다.
  • Pomodoro 25/5가 항상 우월하지 않다는 연구는 사용자 상태와 맥락을 고려할 필요성을 보여준다.
  • micro-break 연구는 짧은 휴식이 피로 감소와 활력 유지에 도움 될 수 있음을 보여준다.
  • 따라서 Focus Orbit은 고정 시간을 강제하지 않고 Mission Log 기반 조정을 적용한다.

Adaptiveness

Focus Orbit의 적응성 정의

Focus Orbit에서 적응성이란 사용자의 작업 설정과 Mission Log에 따라 집중 시간, 휴식 시간, 반복 세션 수가 조정되는 것이다.

작업 설정

사용자가 작업 이름과 기본 루틴을 직접 생성한다.

기록 저장

완료 여부, 중단, 일시정지, Orbit 정보를 저장한다.

루틴 조정

최근 성공 패턴을 바탕으로 다음 추천값에 반영한다.

Concept

프로젝트 컨셉

  • 하나의 작업은 하나의 Mission이 된다.
  • 집중 시간 동안 우주선이 행성을 공전한다.
  • 완료한 작업은 Mission Log에 저장된다.
  • 누적 집중 기록은 Galaxy Map에 반영된다.

Feature 1

Mission Setup

사용자 작업 생성

고정된 작업 유형 대신 사용자가 직접 작업을 추가한다.

루틴 설정

작업 이름, 집중 시간, 휴식 시간을 설정한다.

저장 후 선택

저장된 작업을 클릭하면 미션 시작이 가능하다.

맞춤형 시작점

초기값부터 사용자의 실제 작업 맥락을 반영한다.

Feature 2

Session Cycle

  • 뽀모도로의 반복 구조를 Orbit으로 표현한다.
  • Orbit 1/4, Orbit 2/4처럼 현재 세션 상태를 표시한다.
  • 집중, 휴식, 다음 집중으로 이어지는 흐름을 구현한다.
  • 마지막 Orbit 이후에는 긴 휴식으로 전환할 수 있다.

Feature 3

Active Timer

우주 배경

메인 화면 전체를 활용해 몰입형 타이머 화면을 구성했다.

궤도 시각화

행성, 궤도, 로켓 이동으로 시간 흐름을 표현한다.

진행도 표시

현재 Orbit 진행도와 전체 Mission 진행도를 함께 보여준다.

집중 모드

Minimal Mode와 Reduce Visual Effects로 화면 부담을 줄인다.

Feature 4

Rest Station

  • 집중 세션 종료 후 휴식 단계로 이동한다.
  • 휴식 시간을 다음 탐사를 위한 정비 시간으로 표현한다.
  • 휴식 종료 후 다음 Orbit으로 자연스럽게 이어진다.
  • 집중과 휴식이 하나의 세션 사이클로 연결된다.

Feature 5

Mission Log & Galaxy Map

Mission Log

완료/중단된 미션의 작업 이름, 집중 시간, 휴식 시간, Orbit 정보를 저장한다.

Galaxy Map

완료한 작업을 별처럼 시각화하고 누적 집중 시간에 따라 행성을 해금한다.

기록 기반 추천

저장된 기록은 다음 루틴을 조정하는 데이터가 된다.

성취감

단순 통계가 아니라 우주 탐사 진행도로 집중 기록을 보여준다.

Status Delivery

화면을 보지 않아도 알 수 있는 타이머

브라우저 탭 제목

남은 시간과 현재 상태를 탭 제목에 표시한다.

사운드 알림

집중/휴식 종료 시 소리로 전환 시점을 알려준다.

데스크톱 알림

브라우저 알림을 통해 화면 밖에서도 상태를 전달한다.

Minimal Mode

실제 집중 중에는 큰 타이머와 핵심 조작만 남긴다.

Development

개발 과정

  • Git 브랜치 기반으로 기능 단위 작업을 진행했다.
  • Frontend와 Backend 역할을 분리해 협업 구조를 잡았다.
  • React, TypeScript, Vite 기반으로 프론트엔드를 구현했다.
  • localStorage 기반으로 Mission Log와 설정 상태를 저장했다.
  • 추후 백엔드 API 연동이 가능하도록 기능 단위를 분리했다.
